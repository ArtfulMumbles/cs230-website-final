Adding Items

ExampleMod

If you have followed instuctions up to this point, you should already have everything you need installed. To start, import the example mod project that was installed with Minecraft Forge into Eclipse. You should be able to run this file and open the ExampleMod in the Minecraft Forge client by pressing the "run" button inside of Eclipse. If you have trouble with this, go back to previous steps and follow the instructions carefully.

If the ExampleMod does load correctly, we are ready to get started with adding custom items to the game.

Extending Item Class

In the CustomItem class, extend the Minecraft Item class. This class will be that of our customized item. In the constuctor, call the parent constuctor with super(). We can then move on to changing item specific properties. For example, this.setMaxDamage(0); will change the maximum amount of damage the item can do to 0. Another example could be this.setCreativeTab(CreativeTabs.MISC) which will change the location of the object in the creative menu. There are a bunch of other changes you can make, for a full list, reference the Minecraft Forge documentation. When you're done, you should have something that looks like this:

Adding Item Model

In order to give the item a model (a texture), we need to create a custom JSON file for it. Create a file called my_item.json and save it to the directory src\main\resources\assets\example\models\item\ where example is the modID (in this case, ExampleMod unless otherwise changed). You can then upload a png of your item texture to src\main\resources\assets\example\textures\items\. The inside of your .json file should look something like this:

After this, there is one final step before your custom item can appear in-game.

Registering Items

In order for Minecraft to understand that an object is in the game, it must be registered. this should be done in the ExampleMod class. Item item = new CustomItem();
String registryname = "my_item";
item.setRegistryName(registryName);
item.setUnlocalizedName(item.getRegistryName().toString());
GameRegistry.register(item); In addition, we also need to tell Minecraft the location of our textures. This can be done with these lines of code, also in the ExampleMod class, pre-init. final ModelResourceLocation fullModelLocation = new ModelResourceLocation(item.getRegistryName().toString(), "inventory");
ModelBakery.registerItemVariants(item, fullModelLocation);
ModelLoader.setCustomMeshDefinition(item, new ItemMeshDefinition() {
public ModelResourceLocation getModelLocation(ItemStack stack)
{
return fullModelLocation;
}
});

If you followed all of these steps correctly, you should be able to see a custom item in the game with the texture and name you assigned to it.

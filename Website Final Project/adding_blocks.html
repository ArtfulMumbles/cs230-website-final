In this case, we are using a parent of block/cube_all, which means that the texture we provide will be used on all sides of our block. We can use other default models as well such as:

  • block/cube (all faces are different)
  • block/cube_bottom_top (think wood block)
  • block/orientable (block with variable orientation like a chest)
  • block/cross (tall grass, mushrooms)
  • block/crop (wheat, potatoes)

Each of these parents requires specification by ID in the textues, and you can check those ids in the parent model if you don't know them. The final step will be to place a texture into src\main\resources\assets\example\textures\blocks\ that shares a name with you block. In this example, decorative.png.
